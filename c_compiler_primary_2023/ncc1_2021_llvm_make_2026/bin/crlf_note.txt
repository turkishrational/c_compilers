If files are saved with new lines ending in CRLF 
(for reading in Wordpad and then Notepad), 
CPP gives an error. 
It works correctly if saved with Unix text format
(new line = LF). 
The CPP.EXE file here will be adapted for CRLF later
(TRDOS 386 uses CRLF. When adapting the code
for TRDOS 386, CRLF should be expected 
at the end of lines where LF is expected).
